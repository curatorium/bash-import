# bash-import

Dependency manager and source inliner for bash scripts. Downloads remote bash script dependencies into `.deps/` and can inline them to produce self-contained single-file artifacts.

## Architecture

Single-file tool (`bash-import`) with three subcommands:

- **fetch** -- Scans a script for `source .deps/@github/...` or `source .deps/@https/...` lines, downloads the remote files into `.deps/` using `curl -z` (conditional fetch by mtime).
- **pack** -- Reads a script, replaces every `source .deps/...` line with the inlined content of that dependency. Strips shebang and copyright headers from deps. Injects full LICENSE text when line 2 is a `# Copyright` line. The `.deps/@help` generator is special-cased: executed and wrapped in a heredoc.
- **install** -- Runs pack, then `sudo install -m 755` to `/usr/local/bin`.

All three accept input via file arg, stdin redirect, or pipe. Pack and install accept `-o` for output path.

### Dependency resolution

Dependencies live in `.deps/` using a path-as-URL convention:

```
source .deps/@github/<vendor>/<repo>@<version>/<file>   # GitHub release or tag
source .deps/@https/<host>/<path>                        # Raw HTTPS URL
source .deps/<local-file>                                # Local (committed) dep
```

GitHub deps try release download URL first, then `raw.githubusercontent.com` tag URL as fallback. The `@latest` version resolves via GitHub's latest release redirect.

### Source line matching

Pack recognizes `source` statements (not `.` builtin) that reference `.deps/`. Supports: unquoted, double-quoted, single-quoted, `./`-prefixed, indented, semicolon-terminated. Rejects: commented lines, lines inside string literals.

## External dependency

One remote dependency: `bash-args` from `curatorium/bash-args` (arg parsing: `args:sub`, `args:flag`, `args:opt`, `args:arg`). Fetched into `.deps/@github/curatorium/bash-args@latest/bash-args.sh`.

## Conventions

- **Doc-comments**: Functions are annotated with `@name`, `@type`, `@desc`, `@usage`, `@arg`, `@flag`, `@opt`. These are parsed by `docs.sh` and used to generate both `--help` output and `README.md`.
- **Namespacing**: Functions use `prefix:name` convention (e.g., `import:fetch`, `strict:on`, `docs:tag`).
- **Dynamic scoping**: Some variables (e.g., `quiet`) are set in `import:main` and read in child functions like `import:log`.
- **Test structure**: Tests follow `test:<tier>:<feature>` naming. Tiers: `smoke`, `feature`, `accept`, `regress`. Each test file defines `setup()`, individual `test:*` functions, and an `output()` function that produces a markdown report.
- **README.md is generated**: Never edit it directly. It is produced by `.deps/@readme` and validated by the pre-commit hook.

## Development workflow

```bash
# Fetch remote deps
./bash-import fetch bash-import

# Run tests
./bash-test bash-import.test

# Run full pre-commit checks (shellcheck + tests + README freshness)
.githooks/pre-commit

# Pack into single file
./bash-import pack bash-import -o .dist/bash-import

# Install locally
./bash-import install bash-import
```

## Build and release

Tags matching `v*` trigger `.github/workflows/release.yml` which:
1. Installs `bash-import` and `bash-test` from latest release (bootstrapping).
2. Packs `bash-import` and `bash-test` into `.dist/`.
3. Copies `.deps/*.sh` libs into `.dist/`.
4. Runs pre-commit checks.
5. Creates a GitHub release with all `.dist/` artifacts.
